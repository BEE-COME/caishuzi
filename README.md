## 🚀 快速开始

### 第一步：准备游戏

1. 打开 `index.html` 文件（在浏览器中直接打开即可）
2. 设置你的**秘密数字**（4位数字，如 1234）
3. 选择**先后手**：
   - **我先猜**：你先猜对方的数字（主动进攻）
   - **对方先猜**：对方先猜你的数字（被动防守）

### 第二步：开始对战

点击"开始游戏"按钮进入主界面。

## 🎮 游戏规则

### 基本规则

- 双方各设定一个 **4位数字**（0000-9999）
- 轮流猜测对方数字
- 每次猜测后，对方会告诉你**位置相同的数字个数**（A值）
- **先猜中对方数字（A=4）者获胜**

### 举例说明

```
你的秘密数字：1234
对方猜：1243
你的回答：A=2（只有1和2位置正确）

对方猜：1234
你的回答：A=4（对方获胜！）
```

## 🧠 算法策略图解

### 核心算法原理

工具使用**候选池过滤 + 信息熵最大化**策略：

```
初始状态：10000个候选数字（0000-9999）
    ↓
第1次猜测：1234 → 对方反馈 A=2
    ↓
算法过滤：移除所有与1234匹配数≠2的数字
    ↓
剩余候选：约800-1200个（视反馈而定）
    ↓
算法选择：选择能将候选池分割最均匀的数字
    ↓
重复上述过程，直到A=4获胜
```

### 信息熵策略

算法会优先选择能将候选池**最均匀分割**的数字：

```
候选池1000个时，猜"5678"的预期分割：
  A=0: 200个  A=1: 400个  A=2: 300个  A=3: 80个  A=4: 20个
  信息熵高，能有效缩小范围
```

### 算法优势

| 策略 | 平均猜测次数 | 说明 |
|------|-------------|------|
| 随机猜测 | ~500次 | 无策略，纯运气 |
| 简单排除 | ~50-100次 | 逐个排除 |
| 本工具算法 | **5-15次** | 智能过滤 + 信息熵 |

**精确打击**：
- 算法会直接推荐候选池中的数字
- 此时成功率很高

#### 4. 终局策略（候选池<10）

**逐个验证**：
- 依次尝试剩余候选
- 必定获胜

## ❓ 常见问题

### Q1: 算法建议数字会重复吗？

**A**: 不会。算法会避免推荐历史中已猜测的数字。

### Q2: 为什么有时候建议数字变化？

**A**: 这是正常的。算法根据：
- 历史反馈
- 剩余候选池大小
- 信息熵计算

动态调整最优猜测。

### Q3: 对方说我猜的数字不符合之前的反馈？

**A**: 可能原因：
1. 对方记错了A值
2. 你输错了反馈
3. 对方秘密数字变了

**解决方法**：
- 使用"导出记录"功能核对
- 必要时重新开始游戏

### Q4: 候选池显示0但还没猜中？

**A**: 说明历史记录有矛盾。请：
1. 检查所有反馈是否正确
2. 使用"导出记录"回顾
3. 重新开始游戏

### Q5: 可以同时辅助多局游戏吗？

**A**: 当前版本支持单局游戏。如需多局，可以：
- 在不同浏览器窗口打开
- 或等待后续版本更新

### Q6: 移动端使用有什么注意事项？

**A**:
- 横屏体验更佳
- 左右滑动切换记录标签
- 双击数字快速复制
- 使用数字键盘输入

### Q7: 数据会保存吗？

**A**: 工具会自动保存到浏览器本地存储（localStorage）。关闭后重新打开可继续游戏。

### Q8: 如何彻底清除数据？

**A**:
1. 点击"重新开始"
2. 或清除浏览器缓存
